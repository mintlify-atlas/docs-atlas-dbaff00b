## Overview

Codemogger supports **14 programming languages** through tree-sitter grammars. Each language is configured to extract specific code constructs (functions, classes, types, etc.) and handle large items appropriately.

<Info>
  All language parsers run locally via WebAssembly (WASM) with no external dependencies.
</Info>

### Rust

<ParamField path="Extensions" type="string[]">
  .rs
</ParamField>

<ParamField path="Top-level nodes" type="string[]">

  - `struct_item` - Struct definitions

## Chunking Behavior

### Top-level Extraction

Codemogger extracts top-level definitions from each file:

<Steps>
  <Step title="Parse file">
    Parse file with tree-sitter to generate AST
  </Step>
  <Step title="Extract nodes">
    Extract all top-level nodes matching the language configuration
  </Step>
  <Step title="Generate chunks">
    Create a chunk for each extracted node with metadata
  </Step>
</Steps>

### Large Item Splitting

Items exceeding **150 lines** are split into sub-items:

<Tabs>
  <Tab title="Before Splitting">
    ```rust
    // 250-line impl block
    impl LargeStruct {
        fn method_1() { ... }  // lines 1-50
        fn method_2() { ... }  // lines 51-100
        fn method_3() { ... }  // lines 101-150
        fn method_4() { ... }  // lines 151-200
        fn method_5() { ... }  // lines 201-250
    }
    ```
  </Tab>
  <Tab title="After Splitting">
    ```rust
    // Split into 5 chunks:
    // Chunk 1: method_1 (lines 1-50)
    // Chunk 2: method_2 (lines 51-100)
    // Chunk 3: method_3 (lines 101-150)
    // Chunk 4: method_4 (lines 151-200)
    // Chunk 5: method_5 (lines 201-250)
    ```
  </Tab>
</Tabs>

<Note>
  Only splittable nodes (classes, impl blocks, modules, etc.) are split. Simple functions always remain intact.
</Note>

## Adding New Languages

To add support for a new language, you need:

1. **Tree-sitter grammar** (WASM build)
2. **Language configuration** defining top-level and splittable nodes
3. **Extension mapping** to detect files

<CodeGroup>
```typescript Language Configuration
const NEW_LANG: LanguageConfig = {
  name: "mylang",
  extensions: [".mylang"],
  wasmPath: require.resolve("tree-sitter-mylang/tree-sitter-mylang.wasm"),
  topLevelNodes: [
    "function_declaration",
    "class_declaration",
  ],
  splitNodes: ["class_declaration"],
}
```

```typescript Register Language
const LANGUAGES: LanguageConfig[] = [
  RUST,
  JAVASCRIPT,
  TYPESCRIPT,
  // ... other languages
  NEW_LANG,  // Add your language
]
```
</CodeGroup>

<Tip>
  Check the tree-sitter documentation for your language to identify the correct AST node types.
</Tip>

## Language Detection

Languages are automatically detected by file extension:

```typescript
// Automatic detection
const lang = detectLanguage("src/main.rs")  // Returns Rust config
const lang = detectLanguage("app.tsx")      // Returns TSX config
const lang = detectLanguage("script.py")    // Returns Python config
```

<Warning>
  Files without recognized extensions are skipped during indexing.
</Warning>
