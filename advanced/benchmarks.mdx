## Performance Benchmarks

### Search Performance Comparison

Comparison of semantic search, keyword search, and ripgrep across 4 real-world projects:

| Project | Language | Files | Semantic | Keyword | ripgrep |
|---------|----------|------:|---------:|--------:|--------:|
| [Turso](https://github.com/tursodatabase/turso) | Rust | 748 | 35 ms | 1 ms | 25 ms |
| [Bun](https://github.com/oven-sh/bun) | Zig | 9,255 | 137 ms | 2 ms | 166 ms |
| [TypeScript](https://github.com/microsoft/TypeScript) | TypeScript | 39,298 | 242 ms | 4 ms | 1,500 ms |
| [Kubernetes](https://github.com/kubernetes/kubernetes) | Go | 16,668 | 617 ms | 12 ms | 731 ms |

<Info>
  Keyword search is **25-370x faster than ripgrep** and returns precise definitions instead of thousands of file matches.
</Info>

### Performance Insights

<AccordionGroup>
  <Accordion title="Why is keyword search faster than ripgrep?">
    - **Indexed search**: Pre-built FTS index vs. scanning all files
    - **Structured chunks**: Search only definitions, not entire files
    - **Optimized queries**: Weighted fields prioritize relevant results
    - **No file I/O**: All data in memory-mapped SQLite database
  </Accordion>
  
  <Accordion title="When is semantic search worth the extra time?">
    Semantic search excels when:
    - You don't know exact identifiers
    - Searching by concept or functionality
    - Exploring unfamiliar codebases
    - Finding similar implementations
  </Accordion>
  
  <Accordion title="Why does TypeScript show the biggest ripgrep slowdown?">
    - **Large file count**: 39K files vs 748-16K for other projects
    - **Verbose syntax**: More tokens to scan per file
    - **Many matches**: Common keywords appear in thousands of files
  </Accordion>
</AccordionGroup>

## Benchmark Reproduction

You can reproduce these benchmarks on your own machine:

<CodeGroup>
```bash Index Projects
# Clone and index each project
git clone https://github.com/tursodatabase/turso
codemogger index turso

git clone https://github.com/oven-sh/bun
codemogger index bun

git clone https://github.com/microsoft/TypeScript
codemogger index TypeScript

git clone https://github.com/kubernetes/kubernetes
codemogger index kubernetes
```

```bash Run Searches
# Semantic search
time codemogger search "write-ahead log replication" --mode semantic

# Keyword search
time codemogger search "LogicalLog" --mode keyword

# Compare with ripgrep
time rg "write-ahead log" turso/
```
</CodeGroup>

<Note>
  Actual times may vary based on your hardware, but relative performance should be similar.
</Note>
