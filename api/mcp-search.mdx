## Overview

The `codemogger_search` tool allows AI coding agents to search through indexed codebases efficiently. It supports two search modes:

- **Semantic mode**: Natural language queries using vector embeddings (e.g., "how does authentication work?")
- **Keyword mode**: Precise identifier lookup using full-text search (e.g., "BTreeCursor", "handleRequest")

This tool should be used **first** when exploring or navigating a codebase, before falling back to grep or glob operations.

## Tool Name

```
codemogger_search
```

## Input Schema

<ParamField path="query" type="string" required>
  The search query. Use natural language for semantic mode ("how is user authentication handled?") or specific identifiers for keyword mode ("AuthService", "login").
</ParamField>

<ParamField path="mode" type="enum" default="semantic">
  Search mode to use:
  - `semantic`: For conceptual queries and exploring how features work
  - `keyword`: For exact identifier lookup (functions, classes, types, variables)
</ParamField>

<ParamField path="limit" type="number" default={10}>
  Maximum number of results to return. Must be between 1 and 50.
</ParamField>

<ParamField path="includeSnippet" type="boolean" default={true}>
  Whether to include the full source code snippet in results. When true, eliminates the need for a separate file read operation.
</ParamField>

## Response Format

<ResponseField name="content" type="array">
  Array containing a single text object with formatted search results.
  
  <ResponseField name="type" type="string">
    Always `"text"`
  </ResponseField>
  
  <ResponseField name="text" type="string">
    Formatted search results. Each result includes:
    - File path and line range (e.g., `src/auth.ts:42-58`)
    - Code element kind (function, class, type, etc.)
    - Element name and signature
    - Full code snippet (if `includeSnippet` is true)
  </ResponseField>
</ResponseField>

### No Results Response

If no codebases are indexed:
```
No codebases are indexed yet. Use codemogger_index to index the codebase directory first, then retry this search. Indexing is a one-time operation.
```

If no matches are found:
```
No results found for "<query>" (mode: <mode>). Indexed codebases: <list of indexed paths>
```

## Best Practices for AI Agents

1. **Use semantic mode for exploration**: When you don't know exact names or want to understand how a feature is implemented across multiple files.

2. **Use keyword mode for precision**: When you need to find the definition of a specific function, class, or type.

3. **Set `includeSnippet: true` to avoid extra reads**: This provides the full source code immediately, eliminating the need for separate file read operations.

4. **Start with this tool before grep/glob**: The code index is AST-aware and provides better context than simple text search.

5. **Adjust limit based on task**: Use lower limits (5-10) for targeted searches, higher limits (20-50) when gathering broad context.

## When to Use This Tool

- Finding where a function, class, type, or variable is defined (keyword mode)
- Understanding how a feature or concept is implemented across many files (semantic mode)
- Discovering relevant code when you don't know exact filenames or identifiers (semantic mode)
- Getting quick overviews of code structure without reading entire files

## Prerequisites

The codebase must be indexed first using [`codemogger_index`](/api/mcp-index). If the search returns a "no codebases indexed" message, run the index tool before retrying the search.
